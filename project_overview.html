Excel Deep-Agent

Агент-ассистент консультанта по данным: deep-агент на открытом харнессе DeepAgents (LangChain), заточенный под обработку Excel-файлов — со своим скиллом, субагентом-чистильщиком и мини-бенчем с детерминированной проверкой.
Харнесс: DeepAgents 0.6 Модель: Qwen / GLM через OpenRouter Python 3.11+ Бенч: 8 задач · 3 профиля Лицензия скилла: MIT

1Идея

Технические фичи к агентским харнессам сегодня собираются «из готовых кирпичиков» за часы — ценность в привязке к реальной работе. Этот проект приземляет персонального агента на конкретную специфику: консалтинг по данным и бесконечные эксельки — чистка битых столбцов, дедупликация, объединение списков, своды по регионам, аккуратное сохранение форматирования.

Основные функции: 1) стандартная работа с файлом — формулы, столбцы, цвета и шрифты, содержимое и вид ячеек; 2) анализ данных через pandas (включая текстовые столбцы); 3) причесывание данных — типы, битые ячейки и ошибки, пункты/подпункты, выравнивание по размерам; 4) графики внутренними средствами Excel (нативные диаграммы openpyxl.chart); 5) построчная LLM-обработка ячеек — например, тональность каждого отзыва (тул llm_process_column).

Ключевой принцип: агент не загружает таблицы в контекст LLM. Он работает программно — pandas/openpyxl через shell: разведка через head(), точечные скрипты, проверка результата кодом.

2Архитектура

харнесс и LLM скилл и субагент файлы и рабочая папка бенч и проверка

3Компоненты

Агент · agent.py

Сборка через create_deep_agent: системный промпт с правилами работы с таблицами, LocalShellBackend (файловые тулы + shell), скилл и субагент по профилю.

Кастомные тулы · tools.py

inspect_excel — разведка файла вместо чтения целиком; backup_file — копия .bak; llm_process_column — построчная LLM-обработка текстового столбца пакетами в цикле.

Excel-скилл · SKILL.md

Собственная переработка идеи скиллов Anthropic (MIT): рецепты чистки битых чисел, сохранение заливки и ширины столбцов, дедуп, своды, самопроверка результата.

Субагент data-cleaner

Изолированный контекст и узкий промпт: цикл по строкам, учёт нераспознанных ячеек, запись через openpyxl поверх существующей книги.

Мини-бенч · bench/

8 задач (tasks.json) + детерминированные валидаторы. Раннер гоняет профили, пишет таблицу: PASS/FAIL, токены, время, наличие бэкапа.

Демо-данные · data/

Кейс консалтинга: продажи с выручкой в форматах «1 234,50 руб.» и дублями, два списка посещаемости, клиенты с пропусками сегментов.

4Три профиля — что сравниваем

ПрофильСкиллСубагентГипотеза
baseline——голый харнесс: что модель умеет «из коробки»
skill✔—доменные рецепты поднимают качество (формат, бэкапы)
skill_subagent✔✔изоляция грязной чистки экономит контекст и ошибки

Методика защиты — «предложите 2–3 решения, померьте, защищайте лучшее»: процент решённых задач, суммарные токены, время — всё в bench/results/results.md.

6Запуск

# установка (Python 3.11+)
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env        # вписать OPENROUTER_API_KEY

# одна задача со свежими демо-данными
python main.py --fresh --workdir runs/demo "Очисти столбец 'Выручка (грязная)' в sales_2025.xlsx"

# интерактивный чат / другой профиль
python main.py --workdir runs/demo
python main.py --profile baseline --workdir runs/demo "..."

# полный бенч: 3 профиля × 6 задач
python bench/run_bench.py
Безопасность. LocalShellBackend выполняет команды агента на вашей машине без песочницы (virtual_mode=False). Запускайте на доверенных данных; для «боевого» режима заверните запуск в Docker — как делают настоящие бенчи.
