# Скилл: работа с Excel

## Главные правила

1. **Никогда не загружай таблицу целиком в контекст.** Работай программно через
   pandas/openpyxl: `execute`-тулом запускай короткие python-скрипты.
2. **Сначала разведка, потом действия.** Перед любой правкой посмотри структуру
   (`inspect_excel` или pandas: `sheet_names`, `shape`, `dtypes`, `head(5)`).
   Не печатай весь датафрейм.
3. **Бэкап перед изменением.** Перед первой записью в существующий файл — `backup_file`
   (копия `<имя>.bak`). Один раз на файл, не перезаписывай бэкап. Исходник пользователя
   незаметно не менять.
4. **Только абсолютные пути.** В скриптах и shell-командах используй абсолютные пути
   внутри рабочей папки.
5. **Точечные вопросы — программно.** «Что в 5-й строке 3-го столбца» → `df.iloc[4, 2]`.
6. **Проверка результата.** После записи перечитай файл и сверь 2–3 контрольных
   значения программно (размер, сумма столбца, отсутствие дублей). Потом отчитывайся.

## 1. Стандартная работа с файлом: формулы, столбцы, стили

Правки значений, формул и оформления — через openpyxl поверх существующей книги:

```python
from copy import copy
from openpyxl import load_workbook
from openpyxl.styles import Font, PatternFill, Alignment

wb = load_workbook(PATH)              # стили сохраняются
ws = wb["Лист1"]
ws.cell(row=2, column=3, value=42)                       # значение
ws["E2"] = "=C2*D2"                                      # формула Excel (имена функций
                                                         #   всегда английские: SUM, IF)
ws["F1"] = "Итого"                                       # новый столбец: заголовок...
ws["F1"]._style = copy(ws["E1"]._style)                  # ...со стилем соседнего
ws["A1"].fill = PatternFill("solid", fgColor="FFD966")   # цвет ячейки
ws["A1"].font = Font(bold=True, color="FF0000", size=12) # шрифт
ws["A1"].alignment = Alignment(horizontal="center", wrap_text=True)
ws.column_dimensions["A"].width = 18                     # ширина столбца
ws.row_dimensions[1].height = 22                         # высота строки
ws["B2"].number_format = "#,##0.00"                      # формат: '0%', 'DD.MM.YYYY'
wb.save(PATH)
```

Формулы пиши строкой `"=SUM(B2:B10)"` — Excel пересчитает при открытии. Чтобы прочитать
ВЫЧИСЛЕННЫЕ значения формул: `load_workbook(PATH, data_only=True)` (работает, если файл
сохранялся Excel'ем; иначе считай самим pandas).

## 2. Анализ данных: pandas и текстовые ячейки

```python
import pandas as pd
df = pd.read_excel(PATH, sheet_name="Лист1")
df.describe(); df["X"].value_counts(); df.groupby("Регион")["Выручка"].agg(["sum","mean"])
df["Отзыв"].str.len(); df["Отзыв"].str.contains("брак", case=False)   # текстовые столбцы
```

Сводки и агрегации пиши на **новый лист** (`wb.create_sheet("Свод")`) или в новый файл,
не затирая исходные данные.

## 3. Причесывание данных

- **Типы ячеек**: `pd.to_numeric(col, errors="coerce")`, `pd.to_datetime(...)`;
  после — запиши числа числами (не строками) и поставь `number_format`.
- **Битые числа** (`'1 234,50 руб.'`, `'  1 200  '`, `'1234.5'`):

  ```python
  def to_number(x):
      if isinstance(x, (int, float)):
          return float(x)
      s = (str(x).replace("\xa0", " ").replace("руб.", "").replace("руб", "")
           .replace(" ", "").replace(",", "."))
      return float(s.strip(" ."))
  ```

  Смешанные/нераспознаваемые форматы — цикл по строкам, собери список проблемных ячеек,
  обработай отдельно, не отбрасывай молча. Большие чистки делегируй субагенту `data-cleaner`.
- **Ошибки Excel** (`#DIV/0!`, `#N/A`, `#REF!`): найди циклом
  (`isinstance(cell.value, str) and cell.value.startswith("#")`), исправь формулу
  (`=IFERROR(...; 0)`) или значение — по смыслу задачи.
- **Пункты и подпункты**: нумерация «1., 1.1., 1.2.» — отступом
  `Alignment(indent=уровень)` плюс единый формат; уровень определяй по числу точек.
- **Выравнивание по размерам**: автоподбор ширины

  ```python
  for col in ws.columns:
      width = max(len(str(c.value)) for c in col if c.value is not None) + 2
      ws.column_dimensions[col[0].column_letter].width = min(width, 60)
  ```

- **Объединённые ячейки / скрытые строки**: проверь `ws.merged_cells.ranges`,
  `row_dimensions[r].hidden` — предупреди в отчёте, не ломай молча.

## 4. Графики средствами Excel (нативные, редактируемые)

Только `openpyxl.chart` — диаграмма остаётся живым объектом Excel (НЕ картинка matplotlib):

```python
from openpyxl.chart import BarChart, LineChart, PieChart, Reference

chart = BarChart(); chart.title = "Выручка по регионам"
chart.y_axis.title = "Руб."; chart.x_axis.title = "Регион"
data = Reference(ws, min_col=2, min_row=1, max_row=6)      # значения с заголовком
cats = Reference(ws, min_col=1, min_row=2, max_row=6)      # подписи категорий
chart.add_data(data, titles_from_data=True); chart.set_categories(cats)
ws.add_chart(chart, "D2")                                  # якорь — ячейка листа
wb.save(PATH)
```

Типы: `BarChart` (столбцы; `chart.type="bar"` — горизонтальные), `LineChart`, `PieChart`,
`ScatterChart`, `AreaChart`. Данные для графика должны лежать на листе — если их нет,
сначала запиши свод, потом строй по нему Reference.

## 5. Построчная LLM-обработка ячеек

Когда нужно обработать КАЖДУЮ строку текстового столбца (тональность отзывов,
классификация, извлечение сущностей) — НЕ читай столбец в свой контекст и не оценивай
«в общем». Вызови тул `llm_process_column`: он сам циклом пакетами прогонит каждую
ячейку через LLM и запишет результат в новый столбец.

Пример: тональность отзывов →
`llm_process_column(path, sheet="Отзывы", source_column="Отзыв", new_column="Тональность",
instruction="Определи тональность отзыва. Ответь одним словом: позитивный или негативный")`.

После — проверь выборочно 2–3 строки и `value_counts()` нового столбца.

## Дубликаты и объединение файлов

- Дубли: `df.drop_duplicates()`; по ключу — `subset=[...]`.
- Объединение списков: `pd.concat([df1, df2]).drop_duplicates(subset=["ФИО"])`, сортировка.
- pandas `to_excel` **уничтожает** форматирование — поверх оформленного файла только openpyxl.

## 6. Формулы вместо хардкода (главный принцип «живой» таблицы)

Если значение можно выразить формулой Excel — пиши формулу, а не посчитанное в Python число.
Тогда таблица остаётся пересчитываемой при изменении входов.

```python
# ПЛОХО: посчитали в Python и вписали готовое число
ws["B10"] = float(df["Продажи"].sum())          # застывшее 5000

# ХОРОШО: пусть считает сам Excel
ws["B10"] = "=SUM(B2:B9)"                         # пересчитается при правке данных
ws["C5"] = "=(B5-B2)/B2"                          # темп роста формулой, не 0.15
```

Исключение — результат именно «чистки» исходных данных (распознанные числа из грязного
текста): их записывают значениями. Производные показатели (итоги, доли, темпы) — формулами.

## 7. Профессиональное оформление (требования к любому .xlsx на выходе)

- **Единый шрифт.** Один аккуратный шрифт по всему файлу (например, Calibri/Arial),
  если пользователь не просил иного. Не смешивай шрифты случайно.
- **Ноль формульных ошибок.** В готовом файле не должно остаться `#REF!`, `#DIV/0!`,
  `#VALUE!`, `#N/A`, `#NAME?`. Защищай деления и поиски: `=IFERROR(A2/B2; 0)`,
  `=IFERROR(VLOOKUP(...); "")`. После сохранения перечитай и убедись, что ошибок нет.
- **Уважай существующий шаблон.** Если правишь оформленный файл — изучи и точно повтори
  его форматы, стили, конвенции. Не навязывай своё оформление поверх устоявшегося:
  правила шаблона важнее общих рекомендаций.
- **Заголовки с единицами.** В шапку выноси единицы измерения: «Выручка ($, млн)»,
  «Ставка, ₽/час» — чтобы число читалось однозначно.

## 8. Финмодели: цветовые конвенции и форматы чисел (стандарт индустрии)

Это общепринятые в финансовом моделировании договорённости (им много лет) — применяй,
если пользователь или существующий шаблон не диктуют другое.

**Цвет шрифта = тип содержимого ячейки** (помогает читателю отличить вводы от расчётов):

| Цвет шрифта | RGB | Что означает |
|---|---|---|
| Синий | `0000FF` | Жёстко заданные вводы/допущения, которые меняют под сценарий |
| Чёрный | `000000` | Все формулы и расчёты |
| Зелёный | `008000` | Ссылки на ячейки **других листов** той же книги |
| Красный | `FF0000` | Ссылки на **внешние** файлы |

```python
from openpyxl.styles import Font
ws["B5"].font = Font(color="0000FF")   # ввод-допущение
ws["B6"].font = Font(color="000000")   # =B5*(1+$C$2) — формула
```

**Допущения — в отдельные ячейки, в формулах только ссылки** (не зашивай числа внутрь):

```python
ws["C2"] = 0.05                          # допущение «рост 5%» — отдельной ячейкой (синим)
ws["B6"] = "=B5*(1+$C$2)"                # формула ссылается на допущение, не на 1.05
```

**Форматы чисел** (`cell.number_format`):

| Тип | Формат | Примечание |
|---|---|---|
| Валюта | `"#,##0"` / `"$#,##0"` | единицы — в заголовке столбца |
| Проценты | `"0.0%"` | один знак по умолчанию |
| Мультипликаторы | `"0.0x"` | оценочные (EV/EBITDA, P/E) |
| Нули как «-» | `"#,##0;(#,##0);-"` | нуль показывается прочерком |
| Отрицательные в скобках | `"#,##0;(#,##0)"` | `(123)` вместо `-123` |
| Годы — текстом | заголовок строкой `"2024"` | чтобы не стало `2 024` |

**Документируй хардкоды.** Если число всё же вписано вручную (не формула) — оставь рядом
пометку об источнике: «Источник: [система/документ], [дата], [ссылка]» (соседняя ячейка
или комментарий `ws["B5"].comment = Comment("Источник: …", "agent")`).

## 9. Пересчёт формул без Excel (опционально, LibreOffice)

openpyxl записывает формулы, но НЕ вычисляет их — `data_only=True` вернёт `None`, пока файл
не открыли в Excel/LibreOffice. Если задача требует именно вычисленных значений формул,
пересчитай файл headless-LibreOffice (если он установлен в окружении):

```bash
soffice --headless --calc --convert-to xlsx --outdir <DIR> <FILE>   # пересчитывает и пересохраняет
```

Затем перечитай `load_workbook(PATH, data_only=True)`. Если LibreOffice в окружении нет —
не выдумывай вычисленные значения: оставь формулы и честно отметь это в отчёте.
